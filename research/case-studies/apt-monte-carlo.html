Probabilistic Modeling of APT Cyberattacks: A Monte Carlo Simulation

Modeling discrete random variables in adaptive defense scenarios

Monte Carlo Probability Distribution APT Modeling Cybersecurity

Introduction and Context

This analysis examines the probability distribution of attack scores in a simulated Advanced Persistent Threat (APT) cyberattack scenario with adaptive defense mechanisms. An APT is a sophisticated, multi-stage cyberattack typically conducted by well-funded adversaries such as nation-state actors or organized cybercrime groups. These attacks occur in several phases: initial compromise through social engineering, lateral movement through the network, privilege escalation to gain administrative access, and finally, data exfiltration.

The discrete random variable in this study, denoted as X, represents the total point score an attacker achieves across all four stages of the attack kill chain. This variable was chosen for its complexity and real-world relevance to cybersecurity risk assessment. Unlike basic probability examples involving coin flips or dice rolls, this scenario models conditional probabilities—where outcomes in later stages depend on the success or failure of earlier stages. The inclusion of an adaptive defense system, which strengthens security in response to detected attack activity, creates a dynamic probability environment that reflects actual cybersecurity operations.

Because calculating exact probabilities for this multi-stage conditional scenario is mathematically complex, this analysis employs a Monte Carlo simulation with 1,000 independent trial runs. Monte Carlo simulation is a computational method that uses repeated random sampling to approximate numerical results for complex probabilistic systems. Each trial simulates attacker actions across the four stages—phishing, lateral movement, privilege escalation, and data exfiltration—and the frequencies of each total score are used to build an empirical probability distribution approximating the true theoretical one.

Defining the Discrete Random Variable

A discrete random variable is one whose possible values are countable and distinct, rather than forming a continuous range. For example, the number of heads in three coin flips can only be 0, 1, 2, or 3—never 1.5.

In this study, X is discrete because attack scores are calculated by summing integer point values from discrete events (successful phishing attempts, lateral movements, escalations, etc.). Therefore, X can only take whole-number values within a finite range.

Formal Definition of X

The variable X is defined as:

X = (Stage 1 Score) + (Stage 2 Score) + (Stage 3 Score) + (Stage 4 Score)

where:

  • Stage 1 (S₁): (number of successful phishing clicks) × 2 points per click
  • Stage 2 (S₂): (number of successful lateral movements) × 3 points per movement
  • Stage 3 (S₃): (successful privilege escalations × 5) + 10 bonus points if Domain Admin is achieved
  • Stage 4 (S₄): (successful data exfiltrations × 8) + 15 points for each high-value or critical exfiltration

Because these are integer-based increments (2, 3, 5, 8, 10, 15), X can only be a whole number. It is quantitative because it represents measurable values that can be meaningfully added or averaged.

Range of X

The theoretical sample space of X ranges from 0 to 148 points. The minimum value (X = 0) occurs when the attack fails completely in Stage 1, with all phishing attempts unsuccessful. The theoretical maximum (X = 148) would require perfect success in all stages: S₁ = 12, S₂ = 18, S₃ = 40, and S₄ = 78.

However, not all integer values between 0 and 148 are practically achievable. In the 1,000 Monte Carlo trials, the observed range was 0 to 29 points, showing that adaptive defenses make high scores extremely rare. Scores above 30 are theoretically possible but occur with negligible probability.

The Four Stages of the Attack

Stage 1: Initial Compromise

In Stage 1, the attacker sends phishing emails to six employees in different roles. Each employee has an independent click probability: IT Administrator: 10%, CEO: 15%, Finance: 20%, HR: 30%, Intern: 45%, and Creative Department: 70%. These reflect realistic vulnerability differences—IT admins are trained and cautious, while creative teams interact more with external content.

Because each event is independent, the number of successful phishes follows a Poisson Binomial distribution (a generalization of the binomial distribution for unequal probabilities). Each success grants 2 points. Adaptive defense levels rise as more employees click, with higher alert levels reducing success probabilities in later stages. If no employees click, the attack ends (X = 0). Thus, P(X = 0) represents total failure at Stage 1.

Stage 2: Lateral Movement

Stage 2 models how the attacker moves deeper into the network after gaining initial access. The number of lateral movement attempts depends on Stage 1 success: 1–2 phishes = 3 attempts, 3–4 phishes = 5 attempts, and 5–6 phishes = 6 attempts. Each target system has its own compromise probability (e.g., File Server 40%, Database 30%, Domain Controller 20%). Alert levels reduce these probabilities by 15–30%. Detection during movement raises the alert level again, further lowering success rates. Each successful lateral movement earns 3 points, with a maximum of 18 possible.

Stage 3: Privilege Escalation

Stage 3 represents attempts to gain administrator-level control. The number of escalation attempts equals the number of successful lateral movements. Each system type has a unique escalation probability (File Server 50%, Database 35%, Domain Controller 25%, etc.). If the Domain Controller escalation succeeds, the attacker automatically gains full admin access across all systems and receives a 10-point bonus. Detection probabilities rise with each alert level. Each successful escalation grants 5 points, with a possible total of 40.

Stage 4: Data Exfiltration

In Stage 4, the attacker attempts to steal data. The number of attempts depends on successful escalations from Stage 3, though a minimum of one attempt is allowed even if none succeed. Detection in Stage 3 can limit this to two attempts ("containment countdown"). Exfiltration success probabilities depend on data type: Low-value: 60%, Medium-value: 40%, High-value: 25%, and Critical: 15%. Detection from Data Loss Prevention (DLP) systems scales with alert level. Each successful exfiltration earns 8 points; high-value data earns 15.

Probability Distribution

The probability distribution of X describes how likely each attack score is. Since enumerating all possible paths analytically is infeasible, this study uses the simulation results. Each of the 1,000 trials produced one value of X. The probability P(X = k) was estimated as:

P(X = k) = (Number of trials where X = k) / 1,000

Results show that X = 6 occurred most frequently (P = 0.155). This corresponds to moderate initial compromise and immediate detection. X = 0 occurred in 1.6% of trials, indicating complete failure. Cumulative probability P(X ≤ 10) = 0.722, meaning 72.2% of simulated attacks scored 10 or fewer points. The distribution is right-skewed, concentrated below 15 points, with a long thin tail extending to 29.

Expected Value

The expected value (E[X]) represents the long-run average score:

E[X] = Σ (x × P(X = x))

Using simulation frequencies, E[X] = 8.964 points. Our interpretation of this is that of repeated attacks, the average total score would be about 9 points—just 6.1% of the 148-point maximum. If each point equates to $10,000 in damages, the expected loss per attack is approximately $89,640. Stage contributions average 5 points from Stage 1, 3 from Stage 2, 1 from Stage 3, and nearly 0 from Stage 4. The expected value does not represent the most frequent outcome (mode = 6) but the long-run mean outcome.

Variance and Standard Deviation

Variance (σ²) measures spread around the mean, while standard deviation (σ) is its square root:

σ² = 39.436 points²
σ = 6.280 points

This indicates moderate variability: most scores fall within one standard deviation (approximately 3 to 15 points). About 70% of attacks land in this range, consistent with the empirical rule (68–95–99.7 rule). The coefficient of variation (CV = σ / μ = 70%) confirms moderate relative variability—attacks vary but are not chaotic.

Descriptive Statistics

Key Statistical Measures

  • Mode: 6 points (most common outcome)
  • Median: 7 points (50% of attacks ≤ 7)
  • Quartiles: Q₁ = 5, Q₂ = 7, Q₃ = 12
  • Interquartile Range (IQR): 7 points
  • 90th Percentile: 20 points

About half of all attacks scored between 5 and 12 points, representing partial compromise with early containment. High-damage attacks (≥21) were rare, occurring in only 10% of simulations.

Cumulative Distribution Function (CDF)

The CDF, F(X) = P(X ≤ x), rises steeply from 0 to 0.82 within the first 15 points, showing that most probability mass lies in this low range. Beyond X = 30, the curve flattens, indicating near-zero probability for higher scores. This convex shape (rapid rise, slow tail) is typical of a right-skewed distribution.

No-Defense Comparison

To measure the impact of adaptive defenses, a baseline simulation without defenses was run. Results showed that adaptive defenses reduced average damage by 23.6% and made high scores dramatically rarer. Interestingly, P(X = 0) decreased from 7.4% (no defense) to 1.6% (with defense). Adaptive systems allow minor initial compromises but contain attacks early.

Real-World Implications

Although simplified, this simulation reflects real-world cybersecurity dynamics. Real APTs—such as APT28, APT29, and Lazarus Group—follow similar multi-stage processes. This probability model can be used for risk quantification, translating technical security data into financial terms that decision-makers understand. For instance, reducing the 90th percentile loss from $480,000 to $300,000 through adaptive defenses could justify investments in detection systems or lower-cost cyber insurance.

Strategic Recommendations

Findings indicate that organizations should prioritize:

  1. Strong email security and anti-phishing training (Stage 1–2 transition)
  2. User behavior analytics for detecting lateral movement
  3. Network segmentation to limit internal spread
  4. Fast detection and response over preventing all initial compromises

Preventing all initial compromises is less effective than ensuring fast detection and response to limit total damage.

Conclusion

This model demonstrates how adaptive defenses reshape the probability distribution of cyberattack outcomes. While rare, high-damage events still exist, their likelihood decreases sharply. Most attacks end with minimal impact due to early containment.

Though simplified and based on fixed probabilities, the model effectively illustrates how probabilistic methods and Monte Carlo simulations can quantify cybersecurity performance. It is best viewed as a conceptual framework—one that clarifies where defenses matter most and how mathematical modeling supports real-world risk management.
